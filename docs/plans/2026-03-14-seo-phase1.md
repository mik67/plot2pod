# SEO Phase 1 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add RSS feed, breadcrumb JSON-LD, homepage ItemList JSON-LD, og:locale, and RSS autodiscovery — all without touching the database.

**Architecture:** Pure PHP/markup additions. Each change is isolated to one file. The RSS feed is a new standalone `feed.php`. Structured data blocks are added as inline `<script type="application/ld+json">` tags. RSS autodiscovery and og:locale go into the existing `partials/meta.php` so every page picks them up automatically.

**Tech Stack:** PHP, RSS 2.0, JSON-LD (schema.org)

### Task 2: RSS feed — /feed.php

**Files:**
- Create: `feed.php`

**Step 1: Create feed.php**

```php
<?php
require_once __DIR__ . '/config.php';
require_once __DIR__ . '/db.php';

header('Content-Type: application/rss+xml; charset=utf-8');

$podcasts = $pdo->query(
    "SELECT id, title, description, mp3_path, duration, created_at
     FROM podcasts WHERE published = 1 ORDER BY created_at DESC"
)->fetchAll();

$base = rtrim(SITE_URL, '/');
echo '<?xml version="1.0" encoding="UTF-8"?>' . "\n";
?>
<rss version="2.0">
<channel>
    <title>plot2pod</title>
    <link><?= $base ?>/</link>
    <description>AI-generated debate-format podcasts on any topic</description>
    <language>en-us</language>
    <?php foreach ($podcasts as $p):
        $url    = $base . '/podcast.php?id=' . $p['id'];
        $mp3    = str_starts_with($p['mp3_path'], 'http') ? $p['mp3_path'] : $base . $p['mp3_path'];
        $pubDate = date(DATE_RSS, strtotime($p['created_at']));
    ?>
    <item>
        <title><?= htmlspecialchars($p['title']) ?></title>
        <link><?= $url ?></link>
        <description><?= htmlspecialchars($p['description']) ?></description>
        <pubDate><?= $pubDate ?></pubDate>
        <guid isPermaLink="true"><?= $url ?></guid>
        <enclosure url="<?= htmlspecialchars($mp3) ?>" type="audio/mpeg" length="0"/>
    </item>
    <?php endforeach; ?>
</channel>
</rss>
```

Note: `length="0"` is acceptable for basic RSS; file size is not required.

**Step 2: Test in browser**

Visit `/feed.php` in browser. Confirm valid XML renders with podcast items. Use the W3C Feed Validator (https://validator.w3.org/feed/) if desired.

**Step 3: Commit**

```bash
git add feed.php
git commit -m "feat: add basic RSS 2.0 podcast feed at /feed.php"
```

### Task 4: Homepage ItemList JSON-LD

**Files:**
- Modify: `index.php`

**Step 1: Add JSON-LD block in `<head>` of index.php**

After the `<link rel="stylesheet">` line (line 17), add:

```php
    <?php if (!empty($podcasts)): ?>
    <script type="application/ld+json">
    <?= json_encode([
        '@context' => 'https://schema.org',
        '@type'    => 'ItemList',
        'name'     => 'Latest podcasts',
        'url'      => rtrim(SITE_URL, '/') . '/',
        'itemListElement' => array_map(function($p, $i) {
            return [
                '@type'    => 'ListItem',
                'position' => $i + 1,
                'name'     => $p['title'],
                'url'      => rtrim(SITE_URL, '/') . '/podcast.php?id=' . $p['id'],
            ];
        }, $podcasts, array_keys($podcasts)),
    ], JSON_UNESCAPED_SLASHES | JSON_UNESCAPED_UNICODE) ?>
    </script>
    <?php endif; ?>
```

**Step 2: Verify**

Visit homepage, view source, confirm JSON-LD block is present and well-formed. Paste into Google's Rich Results Test if desired.

**Step 3: Commit**

```bash
git add index.php
git commit -m "feat: add ItemList JSON-LD to homepage"
```
