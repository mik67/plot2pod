MCP

Jak to funguje

Plugin registruje MCP (Model Context Protocol) server, který spouští npx @playwright/mcp@latest. Claude pak může volat nástroje tohoto serveru přímo v konverzaci.

# Konfigurace .mcp.json
{"playwright": {"command": "npx", "args": ["@playwright/mcp@latest"]}}

Agenti (6)

📋

code-reviewer

skóre 0–100

Obecná revize kódu. Kontroluje shodu s CLAUDE.md, stylové porušení, detekci bugů a celkovou kvalitu. Problémy 91–100 jsou kritické.

Aktivační fráze:
"Zkontroluj moje změny" / "Review kód před commitem"
Kdy: po napsání / úpravě kódu, před commitem, před PR
💬

comment-analyzer

přesnost

Analyzuje komentáře vs. skutečný kód. Hledá nepřesné, zastaralé nebo zavádějící komentáře. Prevence comment rot a technického dluhu.

Aktivační fráze:
"Jsou komentáře přesné?" / "Zkontroluj dokumentaci"
Kdy: po přidání docs, před finalizací PR s komentáři
🧪

pr-test-analyzer

gaps 1–10

Kontrola pokrytí testy. Hodnotí behavioral vs. line coverage, kritické mezery (10 = kritické), edge cases a chybové podmínky.

Aktivační fráze:
"Jsou testy dostatečné?" / "Pokrývají testy všechny edge cases?"
Kdy: po vytvoření PR, po přidání nové funkcionality
🔇

silent-failure-hunter

závažnost

Lovec tichých selhání. Hledá problémy v catch blocích, nedostatečné error handling, nevhodné fallback chování a chybějící error logging.

Aktivační fráze:
"Zkontroluj error handling" / "Hledej tiché chyby"
Kdy: po implementaci error handling, before PR s catch bloky
🏗️

type-design-analyzer

4 dimenze 1–10

Expert na design typů. Hodnotí enkapsulaci, vyjádření invariantů, užitečnost a vynucení invariantů – každé na škále 1–10.

Aktivační fráze:
"Zkontroluj design typu UserAccount" / "Má tento typ silné invarianty?"
Kdy: při přidání nového typu, refactoringu datových modelů
✨

code-simplifier

zachovává funkci

Zjednodušuje kód bez změny funkcionality. Zaměřuje se na čitelnost, zbytečnou komplexnost, redundantní kód a konzistenci se standardy projektu.

Aktivační fráze:
"Zjednoduš tento kód" / "Udělej to čitelnější"
Kdy: po code review, když kód funguje ale je komplexní

Doporučený review workflow

1 Napiš kód → code-reviewer
2 Oprav problémy + error handling → silent-failure-hunter
3 Přidej testy → pr-test-analyzer
4 Dokumentuj → comment-analyzer
5 Review projde → polish → code-simplifier
6 Vytvoř PR → hotovo

Skill Creator

Plugin pro tvorbu nových skills a iterativní vylepšování existujících. Obsahuje kompletní eval loop: draft → test → review → improve → opakuj. Optimalizuje trigger description pro přesné spouštění.

1 skill
1 skill 3 agenti 7 Python skriptů Anthropic

Základní workflow

Skill creator zjistí, kde se v procesu nacházíš, a pomůže ti pokročit dál. Ať jdeš od nuly nebo máš hotový draft, přizpůsobí se.

1
Capture Intent – co skill dělá, kdy se spouští, formát výstupu
2
Interview & Research – edge cases, formáty, success criteria
3
Napsat SKILL.md – name, description, instrukce
4
Spustit test cases – with-skill + baseline paralelně
5
Eval viewer – HTML prohlížeč výsledků s feedback formulářem
6
Vylepšit skill – na základě feedbacku, opakuj
7
Optimalizovat description – automatický loop pro přesné spouštění
8
Zabalit a distribuovat – vygeneruje .skill soubor

Eval systém

Pro každý test case se spouštějí dva subagenti najednou: jeden with-skill a jeden jako baseline (bez skill nebo stará verze). Výsledky se porovnávají kvantitativně i kvalitativně.

📊 Benchmark tab
Pass rate, čas, počet tokenů pro každou konfiguraci. Mean ± stddev, delta mezi verzemi, analytické poznatky.
👁️ Outputs tab
Prompt + výstup pro každý test case. Feedback textbox s auto-save. "Submit All Reviews" → feedback.json.
# Spuštění eval vieweru
python eval-viewer/generate_review.py <workspace>/iteration-N \
--skill-name "my-skill" --benchmark benchmark.json
# Agregace výsledků
python -m scripts.aggregate_benchmark <workspace>/iteration-N \
--skill-name <name>

Optimalizace description (trigger loop)

Pole description v SKILL.md frontmatter je primárním mechanismem spouštění. Skill creator ho automaticky optimalizuje pomocí iterativního eval loopу.

  1. 1 Vygeneruje 20 eval queries – mix should-trigger / should-not-trigger (realistické, konkrétní, s kontextem)
  2. 2 Zobrazí je v HTML revieweru, uživatel může editovat a potvrdit
  3. 3 Spustí run_loop.py – 60 % train / 40 % held-out test, max 5 iterací, každou query testuje 3× pro reliabilitu
  4. 4 Claude s extended thinking navrhuje vylepšení; vybírá se nejlepší dle test score (ne train – prevence overfittingu)
  5. 5 Výsledný best_description se zapíše zpět do SKILL.md
# Spuštění trigger optimalizace
python -m scripts.run_loop \
--eval-set trigger-eval.json \
--skill-path ./my-skill \
--model claude-sonnet-4-6 \
--max-iterations 5 --verbose

Design principy

CSS proměnné pro konzistenci. Dominantní barvy s ostrými akcenty předčí bojácné, rovnoměrně rozložené palety. Střídání světlých a tmavých témat.
